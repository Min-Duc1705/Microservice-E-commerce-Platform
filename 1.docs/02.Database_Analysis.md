# Phân tích Cơ sở dữ liệu — Toàn bộ Schema & Logic Flow

> Tài liệu này mô tả cấu trúc CSDL hoàn chỉnh của hệ thống Microservice Shop (tính đến thời điểm hiện tại), căn cứ theo `Impl.md` và mã nguồn thực tế.
> **Kiến trúc:** Mỗi Microservice có **Database riêng** (Database-per-Service pattern). Dữ liệu được đồng bộ qua **RabbitMQ Events**.

---

## 📦 1. AuthService Database

**Mục đích:** Xác thực người dùng (Admin/Nhân viên), quản lý phân quyền theo Role.

### Bảng `AppUsers`

| Cột            | Kiểu          | Ràng buộc            | Mô tả                      |
| -------------- | ------------- | -------------------- | -------------------------- |
| `Id`           | `uuid`        | PK                   |                            |
| `Username`     | `varchar`     | IX (non-unique)      | Tên đăng nhập              |
| `Email`        | `varchar`     | IX **UNIQUE**        | Email đăng nhập            |
| `PasswordHash` | `varchar`     | NOT NULL             | Mật khẩu đã băm (BCrypt)   |
| `IsActive`     | `bool`        | DEFAULT true         | Khóa tài khoản = false     |
| `RefreshToken` | `varchar`     | nullable             | JWT Refresh Token hiện tại |
| `RoleId`       | `uuid`        | FK → Roles, SET NULL | Vai trò được gán           |
| `CreatedAt`    | `timestamptz` |                      |                            |
| `UpdatedAt`    | `timestamptz` | nullable             |                            |

### Bảng `Roles`

| Cột           | Kiểu          | Ràng buộc     | Mô tả                |
| ------------- | ------------- | ------------- | -------------------- |
| `Id`          | `uuid`        | PK            |                      |
| `Name`        | `varchar`     | IX **UNIQUE** | VD: `Admin`, `Staff` |
| `Description` | `varchar`     | nullable      |                      |
| `IsActive`    | `bool`        | DEFAULT true  |                      |
| `CreatedAt`   | `timestamptz` |               |                      |
| `UpdatedAt`   | `timestamptz` | nullable      |                      |

### Bảng `Permissions`

| Cột         | Kiểu          | Ràng buộc                 | Mô tả                             |
| ----------- | ------------- | ------------------------- | --------------------------------- |
| `Id`        | `uuid`        | PK                        |                                   |
| `Name`      | `varchar`     |                           | Tên quyền (VD: `View Orders`)     |
| `ApiPath`   | `varchar`     | IX UNIQUE(ApiPath+Method) | Đường dẫn API (VD: `/api/orders`) |
| `Method`    | `varchar`     | IX UNIQUE(ApiPath+Method) | HTTP Method (`GET`, `POST`...)    |
| `Module`    | `varchar`     |                           | Nhóm chức năng (VD: `Orders`)     |
| `CreatedAt` | `timestamptz` |                           |                                   |
| `UpdatedAt` | `timestamptz` | nullable                  |                                   |

### Bảng `RolePermissions` _(Join Table — M-N)_

| Cột            | Kiểu   | Mô tả            |
| -------------- | ------ | ---------------- |
| `RoleId`       | `uuid` | FK → Roles       |
| `PermissionId` | `uuid` | FK → Permissions |

### Bảng MassTransit Outbox _(Tự động tạo bởi MassTransit)_

- `OutboxMessages` — Lưu Event chờ bắn lên RabbitMQ
- `OutboxStates` — Theo dõi trạng thái Outbox từng task
- `InboxStates` — Đảm bảo idempotency phía Consumer (tránh nhận Event 2 lần)

## 🛒 4. OrderService Database

**Mục đích:** Lưu toàn bộ đơn hàng và chi tiết sản phẩm trong đơn. Có bảng `CustomerProfiles` dùng để cache tránh gọi HTTP sang CustomerService.

### Bảng `Orders`

| Cột               | Kiểu            | Ràng buộc                                     | Mô tả                                                               |
| ----------------- | --------------- | --------------------------------------------- | ------------------------------------------------------------------- |
| `Id`              | `uuid`          | PK                                            |                                                                     |
| `CustomerId`      | `uuid`          | IX, nullable, COMPOSITE IX                    | Có thể null nếu mua không cần TK                                    |
| `CustomerName`    | `varchar`       | IX                                            | **Chốt cứng** tên khách lúc đặt                                     |
| `CustomerPhone`   | `varchar`       | IX                                            | **Chốt cứng** SĐT khách lúc đặt                                     |
| `ShippingAddress` | `varchar`       |                                               | Địa chỉ giao hàng                                                   |
| `PaymentMethod`   | `smallint`      | IX                                            | `0=COD`, `1=BankTransfer`, `2=VNPay`, `3=Momo`                      |
| `ShippingFee`     | `decimal(18,2)` | IX                                            | Phí vận chuyển                                                      |
| `Note`            | `varchar`       |                                               | Ghi chú đơn hàng                                                    |
| `Status`          | `smallint`      | IX, COMPOSITE IX(CustomerId+Status+CreatedAt) | `0=New`, `1=Processing`, `2=Shipping`, `3=Completed`, `4=Cancelled` |
| `CreatedAt`       | `timestamptz`   | IX                                            |                                                                     |
| _`SubTotal`_      | _(computed)_    | `[NotMapped]`                                 | = Σ(Quantity × UnitPrice)                                           |
| _`TotalAmount`_   | _(computed)_    | `[NotMapped]`                                 | = SubTotal + ShippingFee                                            |

### Bảng `OrderItems`

| Cột            | Kiểu            | Ràng buộc                 | Mô tả                                              |
| -------------- | --------------- | ------------------------- | -------------------------------------------------- |
| `Id`           | `uuid`          | PK                        |                                                    |
| `OrderId`      | `uuid`          | FK → Orders (CASCADE), IX |                                                    |
| `ProductId`    | `uuid`          | IX                        | ID sản phẩm (tham chiếu, không FK cứng vì khác DB) |
| `ProductName`  | `varchar`       |                           | **Chốt cứng** tên SP lúc mua                       |
| `Quantity`     | `int`           |                           | Số lượng                                           |
| `UnitPrice`    | `decimal(18,2)` |                           | Giá bán tại thời điểm mua                          |
| `UnitCost`     | `decimal(18,2)` |                           | **Giá nhập** tại thời điểm mua _(Mới bổ sung)_     |
| _`TotalPrice`_ | _(computed)_    | `[NotMapped]`             | = Quantity × UnitPrice                             |
| _`TotalCost`_  | _(computed)_    | `[NotMapped]`             | = Quantity × UnitCost                              |
| _`Profit`_     | _(computed)_    | `[NotMapped]`             | = TotalPrice − TotalCost                           |

### Bảng `CustomerProfiles` _(Cache/Materialized View)_

| Cột             | Kiểu          | Mô tả                                      |
| --------------- | ------------- | ------------------------------------------ |
| `Id`            | `uuid`        | Cùng ID với Customer trong CustomerService |
| `FullName`      | `varchar`     | Tên khách hàng                             |
| `Email`         | `varchar`     | Email                                      |
| `LastUpdatedAt` | `timestamptz` | Thời điểm cập nhật gần nhất từ Event       |

## 🔄 7. Logic Flow — Luồng Hoạt Động

### 7.1 Luồng Đăng nhập (AuthService)

```
Angular                        ApiGateway                    AuthService (DB: auth_db)
  │                                │                                │
  ├──POST /api/auth/login──────────►│                                │
  │  {email, password}             ├──────────forward──────────────►│
  │                                │                        Tìm AppUser by Email
  │                                │                        BCrypt.Verify(password, hash)
  │                                │                        Tạo JWT (userId, roleId, claims)
  │                                │                        Lưu RefreshToken → AppUser
  │◄──{token, refreshToken}────────┤◄────────────────────────────── │
  │  lưu vào localStorage          │                                │
```

### 7.2 Luồng Tạo đơn hàng và Thanh toán (Checkout Flow)

```text
Angular ──POST /api/orders──► ApiGateway
  │         {cartItems...}       │
  │                              ▼
                            OrderService
                            ┌─────────────────────────────────────────┐
                            │ 1. Gọi ProductService lấy giá, kiểm tồn │
                            │ 2. Cập nhật Db (Orders, OrderItems)     │
                            │ 3. Publish OrderCreatedEvent            │
                            └─────────────────────────────────────────┘
                                   │
                 ┌─────────────────┼────────────────────────┐
                 ▼                 ▼                        ▼
        ProductService      CustomerService          PaymentService
        Consumer nhận       Consumer nhận            Consumer nhận
        OrderCreatedEvent   OrderCreatedEvent        OrderCreatedEvent
        → Cập nhật DB       → Cập nhật công nợ       → Tạo PaymentTransaction
        → Tạo giao dịch       (nếu là giao hàng COD)   (Status=Pending)
          xuất kho                                   → Sinh ra Link VNPay/Momo
                                                       nếu chọn Online Payment
                 │                 │                        │
                 ▼                 ▼                        ▼
    ReportService Consumer nhận các thay đổi (cập nhật Revenue/StockSnapshot)
```

### 7.3 Luồng Callback Cổng Thanh Toán (VNPay/Momo)

```text
VNPay Server ──POST /webhook/vnpay──► ApiGateway
                                        │
                                        ▼
                                PaymentService
                                ┌─────────────────────────────────────────┐
                                │ 1. Xác thực chữ ký (Checksum)           │
                                │ 2. UPDATE PaymentTransaction (Success)  │
                                │ 3. Publish PaymentCompletedEvent        │
                                └─────────────────────────────────────────┘
                                        │
                                        ▼
                                OrderService Consumer nhận
                                → UPDATE Orders.Status = Processing (đã thanh toán)
```

### 7.4 Luồng Báo Cáo Doanh Thu (Event-Driven Aggregation)

```text
OrderService
  │ Khi Admin chuyển trạng thái thành Completed
  │ UPDATE Orders.Status = 3
  │ Publish OrderCompletedEvent
  │
  ▼
ReportService Consumer nhận
  → Lấy UnitPrice, UnitCost, Quantity từ Event
  → Xác định đúng dòng RevenueReports bằng Date
  → UPDATE Tổng doanh thu += (UnitPrice * Quantity)
  → UPDATE Tổng chi phí += (UnitCost * Quantity)
  → UPDATE Số lượng sản phẩm bán += Quantity
```

## 🗺️ 8. Sơ đồ Quan hệ Tổng thể (Cross-Service)

```text
╔══════════════════════╗                     ╔════════════════════════════╗
║    AuthService DB    ║                     ║    PaymentService DB       ║
║  AppUsers            ║                     ║  PaymentTransactions       ║
║  Roles, Permissions  ║                     ╚════════════════════════════╝
╚══════════════════════╝                                ▲
                                                        │ OrderCreatedEvent
╔═══════════════════════════════╗    Event    ╔═════════┴═════════════════╗
║     OrderService DB           ║◄────────────║    ProductService DB      ║
║  Orders                       ║ HTTP (read) ║  Categories               ║
║    └─ OrderItems (UnitCost)   ║────────────►║  Products                 ║
║  CustomerProfiles (cache)     ║             ║    └─ InventoryTrans.     ║
╚═══════════════════════════════╝             ╚════════════════════════════╝
   │           │ OrderCreatedEvent                      │
   │           └──────────────────────┐                 │ ProductStockChangedEvent
   │ OrderCompletedEvent              ▼                 │
   │                        ╔════════════════════════════╗
   │                        ║     CustomerService DB     ║
   │                        ║  Customers                 ║
   │                        ║    └─ DebtTransactions     ║
   │                        ╚════════════════════════════╝
   │
   │   (Tổng hợp doanh thu & tồn kho)
   └───────────────────────────► ╔════════════════════════════╗
      (ProductStockChanged)      ║      ReportService DB      ║
      ─────────────────────────► ║  RevenueReports            ║
                                 ║  ProductStockSnapshots     ║
                                 ╚════════════════════════════╝
```
