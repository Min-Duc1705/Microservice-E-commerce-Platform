## ❓ Câu hỏi thường gặp trước khi bắt đầu

### "Tại sao cần Enum riêng cho PaymentMethod thay vì dùng string?"

Trong `OrderCreatedEvent`, `PaymentMethod` được gửi dạng **string** (`"COD"`, `"BankTransfer"`) vì Event là dữ liệu liên service — không thể share Enum. Nhưng trong **database** của PaymentService, ta lưu dạng `int` (smallint) thông qua Enum để:

- Tiết kiệm dung lượng DB
- Dễ filter, index
- Tránh lỗi typo

Khi Consumer nhận event → parse string → convert sang Enum → lưu DB.

### "PaymentService có cần Outbox Pattern không?"

**Có.** PaymentService **publish** `PaymentSucceededEvent` khi Admin confirm thanh toán. Nếu không có Outbox:

- DB lưu thành công nhưng RabbitMQ chết → event mất → OrderService không biết → đơn mãi ở `New`

→ Cần `MassTransit.EntityFrameworkCore` + cấu hình Outbox trong `Program.cs`.

### "Tại sao Consumer inject PaymentDbContext trực tiếp thay vì Repository?"

Consumer chỉ cần 1 thao tác đơn giản: kiểm tra tồn tại → insert. Dùng DbContext trực tiếp nhanh hơn và không cần Specification phức tạp. Repository + Specification chỉ thực sự có ích khi query phức tạp (filter, paging, join).

## BƯỚC 7 — `Data/PaymentDbContext.cs`

**Lý do**: DbContext là cầu nối giữa C# và PostgreSQL.

**Điểm quan trọng:**

- Có thêm 3 DbSet của MassTransit Outbox (`OutboxMessages`, `OutboxStates`, `InboxStates`) — cần thiết để đảm bảo event không bị mất khi publish `PaymentSucceededEvent`
- `HasIndex(e => e.OrderId).IsUnique()` — ràng buộc 1 Order chỉ có đúng 1 Payment ở DB level

```csharp
using MassTransit;
using MassTransit.EntityFrameworkCoreIntegration;
using Microsoft.EntityFrameworkCore;
using PaymentService.Models;

namespace PaymentService.Data;

public class PaymentDbContext : DbContext
{
    public PaymentDbContext(DbContextOptions<PaymentDbContext> options) : base(options) { }

    public DbSet<Payment> Payments { get; set; }

    // ── MassTransit Outbox (bắt buộc vì PaymentService publish PaymentSucceededEvent) ──
    public DbSet<OutboxMessage> OutboxMessages { get; set; }
    public DbSet<OutboxState>   OutboxStates   { get; set; }
    public DbSet<InboxState>    InboxStates    { get; set; }

    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        base.OnModelCreating(modelBuilder);

        modelBuilder.Entity<Payment>(entity =>
        {
            entity.HasKey(e => e.Id);
            entity.Property(e => e.Amount).HasColumnType("decimal(18,2)");

            // 1 Order → 1 Payment (đảm bảo ở cả DB level, không chỉ code)
            entity.HasIndex(e => e.OrderId)
                  .IsUnique()
                  .HasDatabaseName("IX_Payments_OrderId");

            entity.HasIndex(e => e.Status)
                  .HasDatabaseName("IX_Payments_Status");

            entity.HasIndex(e => e.Method)
                  .HasDatabaseName("IX_Payments_Method");

            entity.HasIndex(e => e.CreatedAt)
                  .HasDatabaseName("IX_Payments_CreatedAt");

            // Index composite để báo cáo: lấy danh sách thanh toán theo trạng thái + phương thức
            entity.HasIndex(e => new { e.Status, e.Method })
                  .HasDatabaseName("IX_Payments_Status_Method");
        });

        // Đăng ký bảng Outbox của MassTransit (bắt buộc khi dùng Outbox Pattern)
        modelBuilder.AddInboxStateEntity();
        modelBuilder.AddOutboxMessageEntity();
        modelBuilder.AddOutboxStateEntity();
    }
}
```

## BƯỚC 9 — `Specifications/PaymentFilterSpec.cs`

**Lý do có 2 class trong 1 file**:

- `PaymentFilterSpec`: dùng để lấy data (có Paging + OrderBy)
- `PaymentFilterCountSpec`: chỉ đếm tổng số bản ghi (không Paging) — dùng song song để tính `TotalCount` cho phân trang

```csharp
using CommonService.Specifications;
using PaymentService.Models;
using PaymentService.Models.Request;

namespace PaymentService.Specifications;

/// <summary>
/// Spec lọc danh sách Payment theo nhiều điều kiện, có phân trang và sắp xếp.
/// Dùng trong GetPagedPaymentsAsync → gọi ListAsync().
/// </summary>
public class PaymentFilterSpec : BaseSpecification<Payment>
{
    public PaymentFilterSpec(PaymentFilterRequest f)
        : base(p =>
            (!f.Status.HasValue || p.Status == f.Status.Value) &&
            (!f.Method.HasValue || p.Method == f.Method.Value) &&
            (!f.FromDate.HasValue || p.CreatedAt >= f.FromDate.Value) &&
            (!f.ToDate.HasValue   || p.CreatedAt <= f.ToDate.Value))
    {
        // Sắp xếp
        if (f.SortBy == "Amount")
        {
            if (f.IsDescending) AddOrderByDescending(p => p.Amount);
            else                AddOrderBy(p => p.Amount);
        }
        else // Mặc định: CreatedAt
        {
            if (f.IsDescending) AddOrderByDescending(p => p.CreatedAt);
            else                AddOrderBy(p => p.CreatedAt);
        }

        // Phân trang
        ApplyPaging((f.PageNumber - 1) * f.PageSize, f.PageSize);
    }
}

/// <summary>
/// Spec chỉ đếm (không paging) — dùng song song với PaymentFilterSpec.
/// Cùng điều kiện WHERE nhưng không có OrderBy/Skip/Take.
/// </summary>
public class PaymentFilterCountSpec : BaseSpecification<Payment>
{
    public PaymentFilterCountSpec(PaymentFilterRequest f)
        : base(p =>
            (!f.Status.HasValue || p.Status == f.Status.Value) &&
            (!f.Method.HasValue || p.Method == f.Method.Value) &&
            (!f.FromDate.HasValue || p.CreatedAt >= f.FromDate.Value) &&
            (!f.ToDate.HasValue   || p.CreatedAt <= f.ToDate.Value))
    {
    }
}
```

## BƯỚC 13 — `Services/PaymentServiceImpl.cs`

**File phức tạp nhất** — chứa toàn bộ business logic.

**Điểm quan trọng cần hiểu:**

1. **Tại sao publish `PaymentSucceededEvent` trong Service thay vì Controller?**
   Vì publish event là business logic, không phải HTTP concern. Controller chỉ nhận request + trả response.

2. **Tại sao cần `DateTime.SpecifyKind(..., DateTimeKind.Utc)` cho FromDate/ToDate?**
   PostgreSQL với Npgsql bắt buộc DateTime phải có Kind = Utc. Nếu không → lỗi runtime `"timestamp with time zone"`.

3. **Tại sao check `payment.Status == Succeeded` trước khi confirm?**
   Idempotency — nếu Admin bấm Confirm 2 lần (double-click) → lần 2 trả lỗi thay vì publish event 2 lần.

```csharp
using CommonService.Common;
using CommonService.Events;
using CommonService.Exceptions;
using MassTransit;
using PaymentService.Models;
using PaymentService.Models.Request;
using PaymentService.Models.Response;
using PaymentService.Repository.Interface;
using PaymentService.Services.Interface;
using PaymentService.Specifications;
using PaymentService.Utils.Enum;

namespace PaymentService.Services;

public class PaymentServiceImpl : IPaymentService
{
    private readonly IPaymentRepository _paymentRepo;
    private readonly IPublishEndpoint   _publishEndpoint;

    public PaymentServiceImpl(IPaymentRepository paymentRepo, IPublishEndpoint publishEndpoint)
    {
        _paymentRepo     = paymentRepo;
        _publishEndpoint = publishEndpoint;
    }

    // ─── GET (danh sách) ─────────────────────────────────────────────────────

    public async Task<ResultPaginationDto<PaymentResponse>> GetPagedPaymentsAsync(PaymentFilterRequest filter)
    {
        // PostgreSQL yêu cầu DateTimeKind.Utc cho cột "timestamp with time zone"
        if (filter.FromDate.HasValue)
            filter.FromDate = DateTime.SpecifyKind(filter.FromDate.Value, DateTimeKind.Utc);
        if (filter.ToDate.HasValue)
            filter.ToDate = DateTime.SpecifyKind(filter.ToDate.Value, DateTimeKind.Utc);

        var spec      = new PaymentFilterSpec(filter);
        var countSpec = new PaymentFilterCountSpec(filter);

        var payments   = await _paymentRepo.ListAsync(spec);
        var totalCount = await _paymentRepo.CountAsync(countSpec);

        return new ResultPaginationDto<PaymentResponse>(
            payments.Select(MapToResponse).ToList(),
            filter.PageNumber,
            filter.PageSize,
            totalCount);
    }

    // ─── GET (theo OrderId) ──────────────────────────────────────────────────

    public async Task<PaymentResponse> GetByOrderIdAsync(Guid orderId)
    {
        var spec    = new PaymentByOrderIdSpec(orderId);
        var payment = await _paymentRepo.GetEntityWithSpec(spec)
            ?? throw new NotFoundException($"Không tìm thấy thanh toán cho đơn hàng: {orderId}");

        return MapToResponse(payment);
    }

    // ─── CONFIRM (Admin xác nhận đã nhận tiền) ───────────────────────────────

    public async Task<PaymentResponse> ConfirmPaymentAsync(Guid orderId, ConfirmPaymentRequest request)
    {
        var spec    = new PaymentByOrderIdSpec(orderId);
        var payment = await _paymentRepo.GetEntityWithSpec(spec)
            ?? throw new NotFoundException($"Không tìm thấy thanh toán cho đơn hàng: {orderId}");

        // Guard: tránh confirm lại nếu đã xử lý
        if (payment.Status == PaymentStatus.Succeeded)
            throw new BadRequestException("Đơn hàng này đã được xác nhận thanh toán trước đó.");

        if (payment.Status == PaymentStatus.Failed)
            throw new BadRequestException("Không thể xác nhận thanh toán đã thất bại. Hãy tạo lại đơn.");

        // Cập nhật Payment
        payment.Status          = PaymentStatus.Succeeded;
        payment.TransactionCode = request.TransactionCode;
        payment.Note            = request.Note;
        payment.PaidAt          = DateTime.UtcNow;
        payment.UpdatedAt       = DateTime.UtcNow;

        _paymentRepo.Update(payment);
        await _paymentRepo.SaveChangesAsync(); // ← Lưu DB trước

        // Publish event SAU KHI lưu DB thành công
        // (Outbox Pattern đảm bảo: nếu RabbitMQ chết, event vẫn được gửi lại khi sống)
        await _publishEndpoint.Publish(new PaymentSucceededEvent
        {
            OrderId       = payment.OrderId,
            CustomerId    = payment.CustomerId,
            Amount        = payment.Amount,
            PaymentMethod = payment.Method.ToString(),
            TransactionId = payment.TransactionCode ?? payment.Id.ToString(),
            ProcessedAt   = DateTime.UtcNow
        });

        return MapToResponse(payment);
    }

    // ─── MARK FAILED ─────────────────────────────────────────────────────────

    public async Task MarkPaymentFailedAsync(Guid orderId, string reason)
    {
        var spec    = new PaymentByOrderIdSpec(orderId);
        var payment = await _paymentRepo.GetEntityWithSpec(spec)
            ?? throw new NotFoundException($"Không tìm thấy thanh toán cho đơn hàng: {orderId}");

        if (payment.Status != PaymentStatus.Pending)
            throw new BadRequestException("Chỉ có thể đánh dấu thất bại khi đang ở trạng thái Pending.");

        payment.Status    = PaymentStatus.Failed;
        payment.Note      = reason;
        payment.UpdatedAt = DateTime.UtcNow;

        _paymentRepo.Update(payment);
        await _paymentRepo.SaveChangesAsync();

        await _publishEndpoint.Publish(new PaymentFailedEvent
        {
            OrderId     = payment.OrderId,
            Reason      = reason,
            ProcessedAt = DateTime.UtcNow
        });
    }

    // ─── Helper ──────────────────────────────────────────────────────────────

    private static PaymentResponse MapToResponse(Payment p) => new()
    {
        Id              = p.Id,
        OrderId         = p.OrderId,
        CustomerId      = p.CustomerId,
        Amount          = p.Amount,
        Method          = p.Method.ToString(),
        Status          = p.Status.ToString(),
        TransactionCode = p.TransactionCode,
        PaidAt          = p.PaidAt,
        Note            = p.Note,
        CreatedAt       = p.CreatedAt
    };
}
```

## BƯỚC 14 — `Consumers/OrderCreatedConsumer.cs`

> ⚠️ **Xóa file `OrderCreatedConsumerPayment.cs`** cũ (skeleton rỗng), tạo file mới tên `OrderCreatedConsumer.cs`.

**Lý do đổi tên**: Convention đặt tên Consumer là `{ServiceName}{EventName}Consumer` hoặc đơn giản hơn là `{EventName}Consumer`. File cũ tên `OrderCreatedConsumerPayment` là đặt ngược.

**Flow**: `OrderCreatedEvent` đến → Kiểm tra idempotency → Tạo Payment (Pending).

**Tại sao inject `PaymentDbContext` trực tiếp mà không dùng `IPaymentRepository`?**

Có 2 quan điểm:

- **Repository**: Nhất quán với architecture. Nếu sau này thêm transaction/unit of work → dễ hơn.
- **DbContext trực tiếp** (code này đang dùng): Đơn giản hơn cho Consumer vì logic chỉ là insert 1 bản ghi.

→ **Dự án này chọn Repository** để nhất quán DDD:

```csharp
using CommonService.Events;
using MassTransit;
using PaymentService.Data;
using PaymentService.Models;
using PaymentService.Repository.Interface;
using PaymentService.Specifications;
using PaymentService.Utils.Enum;

namespace PaymentService.Consumers;

/// <summary>
/// Lắng nghe OrderCreatedEvent từ OrderService.
/// Tự động tạo bản ghi Payment với Status = Pending.
/// Admin sẽ xác nhận thanh toán sau qua PUT /payments/order/{orderId}/confirm.
/// </summary>
public class OrderCreatedConsumer : IConsumer<OrderCreatedEvent>
{
    private readonly IPaymentRepository _paymentRepo;

    public OrderCreatedConsumer(IPaymentRepository paymentRepo)
    {
        _paymentRepo = paymentRepo;
    }

    public async Task Consume(ConsumeContext<OrderCreatedEvent> context)
    {
        var ev = context.Message;

        // ── Idempotency check ─────────────────────────────────────────────
        // Nếu event được gửi lại (RabbitMQ retry) → bỏ qua, tránh tạo Payment trùng
        var existing = new PaymentByOrderIdSpec(ev.OrderId);
        var exists   = await _paymentRepo.GetEntityWithSpec(existing);
        if (exists != null) return;

        // ── Parse PaymentMethod từ string event sang Enum ─────────────────
        var method = ev.PaymentMethod switch
        {
            "BankTransfer" => PaymentMethod.BankTransfer,
            _              => PaymentMethod.COD  // COD là default nếu không nhận ra
        };

        var payment = new Payment
        {
            Id         = Guid.NewGuid(),
            OrderId    = ev.OrderId,
            CustomerId = ev.CustomerId,
            Amount     = ev.TotalAmount,
            Method     = method,
            Status     = PaymentStatus.Pending,
            CreatedAt  = DateTime.UtcNow
        };

        await _paymentRepo.AddAsync(payment);
        await _paymentRepo.SaveChangesAsync();
    }
}
```

## BƯỚC 18 — `Program.cs`

**Thay toàn bộ nội dung** file hiện tại (đang là skeleton comment-out):

```csharp
using MassTransit;
using Microsoft.EntityFrameworkCore;
using PaymentService.Consumers;
using PaymentService.Data;
using PaymentService.Repository;
using PaymentService.Repository.Interface;
using PaymentService.Services;
using PaymentService.Services.Interface;

var builder = WebApplication.CreateBuilder(args);

// ── 1. Database ───────────────────────────────────────────────────────────────
builder.Services.AddDbContext<PaymentDbContext>(options =>
    options.UseNpgsql(builder.Configuration.GetConnectionString("Default")));

// ── 2. Repository & Service (Dependency Injection) ────────────────────────────
builder.Services.AddScoped<IPaymentRepository, PaymentRepository>();
builder.Services.AddScoped<IPaymentService, PaymentServiceImpl>();

// ── 3. MassTransit + RabbitMQ + Outbox Pattern ───────────────────────────────
builder.Services.AddMassTransit(x =>
{
    x.SetKebabCaseEndpointNameFormatter();

    // Đăng ký Consumer lắng nghe OrderCreatedEvent
    x.AddConsumer<OrderCreatedConsumer>();

    // Outbox Pattern: đảm bảo PaymentSucceededEvent không bị mất
    // khi PaymentService confirm → lưu event vào DB → gửi sau khi RabbitMQ sẵn sàng
    x.AddEntityFrameworkOutbox<PaymentDbContext>(o =>
    {
        o.UsePostgres();
        o.UseBusOutbox();
    });

    x.UsingRabbitMq((context, cfg) =>
    {
        var host     = builder.Configuration["RabbitMQ:Host"]     ?? "localhost";
        var port     = ushort.Parse(builder.Configuration["RabbitMQ:Port"] ?? "5673");
        var username = builder.Configuration["RabbitMQ:Username"] ?? "guest";
        var password = builder.Configuration["RabbitMQ:Password"] ?? "guest";

        cfg.Host(host, port, "/", h =>
        {
            h.Username(username);
            h.Password(password);
        });

        cfg.ConfigureEndpoints(context);
    });
});

// ── 4. Exception Handler + Response Wrapper ───────────────────────────────────
builder.Services.AddExceptionHandler<CommonService.Exceptions.GlobalExceptionHandler>();
builder.Services.AddProblemDetails();
builder.Services.AddControllers(options =>
    options.Filters.Add<CommonService.Filters.FormatResponseFilter>());

builder.Services.AddEndpointsApiExplorer();
builder.Services.AddSwaggerGen(c =>
    c.SwaggerDoc("v1", new() { Title = "PaymentService API", Version = "v1" }));

var app = builder.Build();

if (app.Environment.IsDevelopment())
{
    app.UseSwagger();
    app.UseSwaggerUI();
}

app.UseExceptionHandler();
app.UseAuthorization();
app.MapControllers();
app.Run();
```
