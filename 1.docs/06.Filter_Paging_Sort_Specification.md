# Filter, Phân trang, Sort & Specification Pattern

> Giải thích cách hệ thống xử lý filter / sort / paging từ Request của client đến SQL thực thi — dựa trên code thực tế của dự án.

---

## 🗺️ Tổng quan luồng

```
Client gửi request
  GET /api/products?searchTerm=áo&status=0&sortBy=price&isDescending=true&pageNumber=2&pageSize=10
        │
        ▼
Controller nhận [FromQuery] ProductFilterRequest
        │
        ▼
Service tạo Specification (ProductFilterSpec)
  → Specification đóng gói: WHERE + ORDER BY + SKIP/TAKE
        │
        ▼
Repository.ListAsync(spec) + CountAsync(countSpec)
  → SpecificationEvaluator dịch Spec → IQueryable → SQL
        │
        ▼
EF Core thực thi SQL → trả về data
        │
        ▼
Service map Entity → DTO → trả về ResultPaginationDto<T>
        │
        ▼
FormatResponseFilter bọc vào ApiResponse<T>
        │
        ▼
Client nhận JSON chuẩn
```

## 1. Các lớp cốt lõi trong CommonService

### 1.1 `ISpecification<T>` — Hợp đồng của mọi Specification

File: `CommonService/Specifications/ISpecification.cs`

```csharp
public interface ISpecification<T>
{
    Expression<Func<T, bool>>?       Criteria           { get; }  // WHERE
    List<Expression<Func<T, object>>> Includes           { get; }  // JOIN (Include)
    Expression<Func<T, object>>?     OrderBy            { get; }  // ORDER BY ASC
    Expression<Func<T, object>>?     OrderByDescending  { get; }  // ORDER BY DESC
    int  Skip            { get; }   // Bỏ qua N bản ghi đầu
    int  Take            { get; }   // Lấy N bản ghi tiếp theo
    bool IsPagingEnabled { get; }   // Có áp dụng paging không?
}
```

**Ý nghĩa**: Interface này giống như một "bản đặc tả truy vấn". Thay vì viết SQL hay LINQ trực tiếp khắp nơi, mọi điều kiện được đóng gói trong 1 object triển khai interface này.

### 1.2 `BaseSpecification<T>` — Class cha, mọi Spec kế thừa từ đây

File: `CommonService/Specifications/BaseSpecification.cs`

```csharp
public abstract class BaseSpecification<T> : ISpecification<T>
{
    // Các property của ISpecification (đọc từ ngoài)
    public Expression<Func<T, bool>>?        Criteria          { get; private set; }
    public List<Expression<Func<T, object>>> Includes          { get; } = new();
    public Expression<Func<T, object>>?      OrderBy           { get; private set; }
    public Expression<Func<T, object>>?      OrderByDescending { get; private set; }
    public int  Skip            { get; private set; }
    public int  Take            { get; private set; }
    public bool IsPagingEnabled { get; private set; }

    // Constructor không có WHERE → lấy tất cả
    protected BaseSpecification() { }

    // Constructor có WHERE clause
    protected BaseSpecification(Expression<Func<T, bool>> criteria)
        => Criteria = criteria;

    // Các phương thức protected để class con gọi trong constructor
    protected void AddInclude(Expression<Func<T, object>> expr)     => Includes.Add(expr);
    protected void AddOrderBy(Expression<Func<T, object>> expr)     => OrderBy = expr;
    protected void AddOrderByDescending(Expression<Func<T, object>> expr) => OrderByDescending = expr;
    protected void ApplyPaging(int skip, int take)
    {
        Skip = skip; Take = take; IsPagingEnabled = true;
    }
}
```

**Điểm quan trọng**:

- `private set` → class con không được gán trực tiếp, phải dùng các method protected như `AddOrderBy()`, `ApplyPaging()` → tránh bug vô tình override
- Constructor của class con gọi `base(criteria)` để set `Criteria`

### 1.3 `SpecificationEvaluator<T>` — Bộ dịch Spec → SQL

File: `CommonService/Specifications/SpecificationEvaluator.cs`

```csharp
public class SpecificationEvaluator<T> where T : class
{
    public static IQueryable<T> GetQuery(IQueryable<T> inputQuery, ISpecification<T> spec)
    {
        var query = inputQuery;

        // Bước 1: Áp WHERE (nếu có Criteria)
        if (spec.Criteria != null)
            query = query.Where(spec.Criteria);

        // Bước 2: Áp JOIN (Include liên kết bảng liên quan)
        query = spec.Includes.Aggregate(query,
            (current, include) => current.Include(include));

        // Bước 3: Áp ORDER BY (ưu tiên ASC, nếu không có thì DESC)
        if (spec.OrderBy != null)
            query = query.OrderBy(spec.OrderBy);
        else if (spec.OrderByDescending != null)
            query = query.OrderByDescending(spec.OrderByDescending);

        // Bước 4: Áp PAGING (Skip & Take)
        if (spec.IsPagingEnabled)
            query = query.Skip(spec.Skip).Take(spec.Take);

        return query; // Vẫn là IQueryable — chưa thực thi, EF Core dịch sang SQL sau
    }
}
```

**Quan trọng**: `GetQuery` trả về `IQueryable<T>` (chưa chạy SQL). SQL chỉ thực sự chạy khi gọi `.ToListAsync()` hoặc `.FirstOrDefaultAsync()` bên trong `GenericRepository`.

```csharp
public async Task<IReadOnlyList<T>> ListAsync(ISpecification<T> spec)
{
    var query = SpecificationEvaluator<T>.GetQuery(_dbSet.AsQueryable(), spec);
    return await query.ToListAsync(); // ← Đây mới thực thi SQL
}

public async Task<int> CountAsync(ISpecification<T> spec)
{
    var query = _dbSet.AsQueryable();
    if (spec.Criteria != null)
        query = query.Where(spec.Criteria); // Chỉ áp WHERE, không Skip/Take
    return await query.CountAsync();
}
```

### 1.5 `ResultPaginationDto<T>` — Cấu trúc response phân trang

File: `CommonService/Common/ResultPaginationDto.cs`

```csharp
public class ResultPaginationDto<T>
{
    public MetaInfo?         Meta   { get; set; }
    public IReadOnlyList<T>? Result { get; set; }

    public ResultPaginationDto(IReadOnlyList<T> items, int page, int pageSize, int totalRecords)
    {
        Meta = new MetaInfo
        {
            Page     = page,
            PageSize = pageSize,
            Total    = totalRecords,
            Pages    = (int)Math.Ceiling(totalRecords / (double)pageSize) // Tổng số trang
        };
        Result = items;
    }

    public class MetaInfo
    {
        public int  Page     { get; set; }
        public int  PageSize { get; set; }
        public int  Pages    { get; set; }  // = ceil(Total / PageSize)
        public long Total    { get; set; }  // Tổng bản ghi (không paging)
    }
}
```

```json
  "data": {
    "meta": {
      "page": 2,
      "pageSize": 10,
      "pages": 5,
      "total": 47
    },
    "result": [ ... ]
  }
}
```

## 2. Specification Pattern trong thực tế — 3 service thực tế

### 2.1 ProductService — `ProductFilterSpec`

File: `ProductService/Specifications/ProductFilterSpec.cs`

```csharp
public class ProductFilterSpec : BaseSpecification<Product>
{
    public ProductFilterSpec(
        string? searchTerm, Guid? categoryId, ProductStatus? status,
        string? sortBy, bool isDescending, int pageNumber, int pageSize,
        DateTime? fromDate = null, DateTime? toDate = null)

        // ── WHERE clause: kết hợp nhiều điều kiện bằng &&
        : base(p =>
            // Tìm kiếm: null/empty → bỏ qua; có giá trị → match Name hoặc SKU
            (string.IsNullOrEmpty(searchTerm) ||
                p.Name.ToLower().Contains(searchTerm.ToLower()) ||
                p.SKU.ToLower().Contains(searchTerm.ToLower())) &&

            // Filter: null → không filter; có giá trị → phải match chính xác
            (!categoryId.HasValue || p.CategoryId == categoryId.Value) &&
            (!status.HasValue     || p.Status     == status.Value)     &&
            (!fromDate.HasValue   || p.CreatedAt  >= fromDate.Value)   &&
            (!toDate.HasValue     || p.CreatedAt  <= toDate.Value.AddDays(1)))
    {
        // JOIN với bảng Categories để lấy CategoryName
        AddInclude(p => p.Category!);

        // ── Dynamic Sort bằng Dictionary ──
        // Thêm cột sort mới = thêm 1 dòng vào đây
        var sortMappings = new Dictionary<string, Expression<Func<Product, object>>>
        {
            ["name"]          = p => p.Name,
            ["sku"]           = p => p.SKU,
            ["price"]         = p => p.Price,
            ["costprice"]     = p => p.CostPrice,
            ["stockquantity"] = p => p.StockQuantity,
            ["status"]        = p => p.Status,
            ["createdat"]     = p => p.CreatedAt,
        };

        var sortKey    = (sortBy ?? "createdAt").ToLower();
        var sortExpr   = sortMappings.GetValueOrDefault(sortKey, sortMappings["createdat"]);

        if (isDescending) AddOrderByDescending(sortExpr);
        else              AddOrderBy(sortExpr);

        // ── Paging ──
        // pageNumber=2, pageSize=10 → Skip=10, Take=10
        ApplyPaging((pageNumber - 1) * pageSize, pageSize);
    }
}
```

**SQL tương đương được EF Core sinh ra:**

```sql
SELECT p.*, c.Name AS CategoryName
FROM Products p
LEFT JOIN Categories c ON p.CategoryId = c.Id
WHERE p.IsDeleted = false                          -- (Global Filter tự động)
  AND (p.Name ILIKE '%áo%' OR p.SKU ILIKE '%áo%') -- searchTerm
  AND p.CategoryId = '...'                         -- categoryId
  AND p.Status = 0                                  -- status
ORDER BY p.Price DESC                              -- sortBy=price, isDescending=true
OFFSET 10 ROWS FETCH NEXT 10 ROWS ONLY;           -- page=2, size=10
```

### Bước 1: Client gửi request

```
GET /api/products?searchTerm=áo&categoryId=abc&status=0&sortBy=price&isDescending=true&pageNumber=2&pageSize=10
```

### Bước 2: Controller nhận qua `[FromQuery]`

```csharp
[HttpGet]
public async Task<ResultPaginationDto<ProductResponse>> GetAll(
    [FromQuery] string? searchTerm,
    [FromQuery] Guid? categoryId,
    [FromQuery] ProductStatus? status,
    [FromQuery] string? sortBy = "createdAt",
    [FromQuery] bool isDescending = true,
    [FromQuery] int pageNumber = 1,
    [FromQuery] int pageSize = 20,
    [FromQuery] DateTime? fromDate = null,
    [FromQuery] DateTime? toDate = null)
    => await _productService.GetPagedProductsAsync(
           searchTerm, categoryId, status, sortBy, isDescending, pageNumber, pageSize, fromDate, toDate);
```

### Bước 3: Service tạo 2 Specification

```csharp
public async Task<ResultPaginationDto<ProductResponse>> GetPagedProductsAsync(...)
{
    // Chuẩn hoá DateTime sang UTC (PostgreSQL yêu cầu)
    if (fromDate.HasValue) fromDate = DateTime.SpecifyKind(fromDate.Value, DateTimeKind.Utc);
    if (toDate.HasValue)   toDate   = DateTime.SpecifyKind(toDate.Value,   DateTimeKind.Utc);

    // Spec LẤY DATA: có WHERE + ORDER BY + SKIP/TAKE
    var spec = new ProductFilterSpec(
        searchTerm, categoryId, status, sortBy, isDescending, pageNumber, pageSize, fromDate, toDate);

    // Spec CHỈ ĐẾM: cùng WHERE nhưng KHÔNG có ORDER BY và KHÔNG có SKIP/TAKE
    var countSpec = new ProductFilterCountSpec(
        searchTerm, categoryId, status, fromDate, toDate);

    // Chạy song song 2 query (hoặc tuần tự)
    var products   = await _productRepo.ListAsync(spec);      // SELECT ... OFFSET ... LIMIT ...
    var totalCount = await _productRepo.CountAsync(countSpec); // SELECT COUNT(*)

    return new ResultPaginationDto<ProductResponse>(
        products.Select(MapToResponse).ToList(),
        pageNumber, pageSize, totalCount);
}
```

**Tại sao cần 2 Spec riêng?**

```
Nếu dùng 1 spec có paging cho CountAsync:
  SELECT COUNT(*) ... OFFSET 10 LIMIT 10
  → Luôn trả về 10 (hoặc ít hơn), không phải tổng số bản ghi!

Phải dùng CountSpec không paging:
  SELECT COUNT(*) FROM Products WHERE ...
  → Trả về đúng tổng số (VD: 47)
```

## 4. Pattern: Tại sao cần cả FilterSpec lẫn FilterCountSpec?

```
┌─────────────────────────────────────────────────────────────────┐
│  ProductFilterSpec (có paging)                                  │
│  → SELECT * FROM Products WHERE ... ORDER BY ... OFFSET LIMIT   │
│  → Trả về đúng trang dữ liệu cần hiển thị                       │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│  ProductFilterCountSpec (không paging)                          │
│  → SELECT COUNT(*) FROM Products WHERE ...                      │
│  → Trả về tổng số bản ghi (để tính Pages = ceil(47/10) = 5)    │
└─────────────────────────────────────────────────────────────────┘
```

**Frontend dùng `meta.pages` và `meta.total` để**:

- Render số trang (pagination component)
- Hiển thị "Đang hiển thị 11-20 trong tổng số 47 sản phẩm"
- Disable nút "Trang sau" khi `page >= pages`

## 5. Dynamic Sort — Cách thêm cột sort mới

Tất cả service dùng cùng pattern `Dictionary<string, Expression>`:

```csharp
var sortMappings = new Dictionary<string, Expression<Func<Product, object>>>
{
    ["name"]      = p => p.Name,
    ["price"]     = p => p.Price,
    ["createdat"] = p => p.CreatedAt,
    // Muốn thêm sort theo UpdatedAt? Chỉ cần thêm dòng này:
    ["updatedat"] = p => p.UpdatedAt!,
};

// Nếu client gửi sortBy không hợp lệ (VD: "xyz") → fallback về "createdat"
var sortKey  = (sortBy ?? "createdAt").ToLower();
var sortExpr = sortMappings.GetValueOrDefault(sortKey, sortMappings["createdat"]);

if (isDescending) AddOrderByDescending(sortExpr);
else              AddOrderBy(sortExpr);
```

**Lợi ích so với `if/else if` chain:**

```csharp
// ❌ Cách cũ — dài, dễ thiếu case, khó thêm mới
if (sortBy == "name")         query = query.OrderBy(p => p.Name);
else if (sortBy == "price")   query = query.OrderBy(p => p.Price);
else if (sortBy == "sku")     query = query.OrderBy(p => p.SKU);
// ... 10 dòng nữa ...

// ✅ Cách dùng — thêm cột mới = thêm 1 dòng vào Dictionary
var sortMappings = new Dictionary<string, Expression<Func<Product, object>>> { ... };
```

## 6. Filter kiểu Optional (`!HasValue` pattern)

Mọi filter trong dự án đều theo pattern: **null = không filter, có giá trị = filter chính xác**.

```csharp
// Dịch nghĩa từng điều kiện:
(!status.HasValue || p.Status == status.Value)

// Nếu status = null  → (!true  || ...) → true  → không lọc
// Nếu status = 0     → (!false || p.Status == 0) → phụ thuộc p.Status == 0
```

**Vì sao dùng `||` thay vì `if` bên trong constructor?**

Vì `Criteria` là `Expression<Func<T, bool>>` — một lambda expression được EF Core dịch sang SQL. Không thể dùng `if/else` thông thường bên trong expression tree. Phải viết điều kiện gộp lại trong 1 lambda.

```csharp
// ❌ Không hoạt động — EF Core không dịch được if/else trong Expression tree
: base(p => {
    if (status.HasValue) return p.Status == status.Value;
    return true;
})

// ✅ Đúng — viết gộp trong 1 expression
: base(p => (!status.HasValue || p.Status == status.Value))
```

## 8. Khi nào KHÔNG dùng Specification?

Specification phù hợp với các query **đọc dữ liệu phức tạp** (filter + sort + paging).

**Không cần dùng khi**:

| Tình huống                        | Cách làm                                                             |
| --------------------------------- | -------------------------------------------------------------------- |
| Lấy 1 bản ghi theo PK             | `repository.GetByIdAsync(id)`                                        |
| Consumer chỉ cần insert 1 bản ghi | Dùng `DbContext` trực tiếp                                           |
| Query phức tạp với GROUP BY, SUM  | Dùng `DbContext.Database.SqlQuery` hoặc LINQ trực tiếp trong Service |
| Kiểm tra tồn tại đơn giản         | `_dbContext.Payments.AnyAsync(p => p.OrderId == id)`                 |

> ReportService là ví dụ tốt: query GROUP BY + SUM để báo cáo doanh thu → dùng EF Core LINQ trực tiếp trong `ReportServiceImpl`, không cần Specification.
