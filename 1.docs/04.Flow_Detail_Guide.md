# 📘 Chi tiết luồng chạy từng flow trong dự án MicroserviceShop

> Mỗi flow bên dưới đều nêu **đoạn code nào chạy → ở file nào → kết quả gì**.
> Ví dụ thực tế dùng khách hàng "Nguyễn Văn A" đặt 2 sản phẩm: Áo (200k) + Quần (300k).

---

## 🔐 FLOW 1 — Đăng ký tài khoản

**Trigger:** `POST /api/v1/auth/register` (qua API Gateway port 8080)

### Bước 3 — AuthServiceImpl xử lý

**File:** `AuthService/Services/AuthServiceImpl.cs` — method `RegisterAsync()`

```csharp
// Kiểm tra email/username trùng
if (await _userRepo.EmailExistsAsync(request.Email))
    throw new BadRequestException($"Email '{request.Email}' đã tồn tại.");

// Tạo AppUser mới, hash password bằng BCrypt
var user = new AppUser
{
    Id           = Guid.NewGuid(),  // → VD: "aaa-111-..."
    Username     = "nguyenvana",
    Email        = "a@gmail.com",
    PasswordHash = BCrypt.HashPassword("123456"),
    IsActive     = true,
    RoleId       = defaultRole.Id   // Role "USER"
};

await _userRepo.AddAsync(user);   // Track entity, CHƯA lưu DB

// ⚡ Outbox Pattern: publish vào Outbox Table (CÙNG transaction với SaveChanges bên dưới)
await _publishEndpoint.Publish(new UserRegisteredEvent
{
    UserId       = user.Id,        // "aaa-111-..."
    Username     = "nguyenvana",
    Email        = "a@gmail.com",
    RegisteredAt = DateTime.UtcNow
});

await _userRepo.SaveChangesAsync(); // ← 1 transaction duy nhất: lưu AppUser + OutboxMessage
```

> **Outbox Pattern đảm bảo:** Dù RabbitMQ có down ngay lúc này, event đã được lưu vào DB.
> Background Worker của MassTransit sẽ gửi lên RabbitMQ sau khi RabbitMQ khôi phục.

### Bước 4 — RabbitMQ phân phối `UserRegisteredEvent`

Event được gửi tới 2 queue song song (mỗi consumer đăng ký 1 queue riêng):

```
UserRegisteredEvent
    ├──▶ [queue: order-user-registered-consumer]   → OrderService
    └──▶ [queue: user-registered-consumer]          → CustomerService
```

---

### Bước 4a — OrderService nhận, tạo CustomerProfile

**File:** `OrderService/Consumers/UserRegisteredConsumer.cs`

```csharp
public async Task Consume(ConsumeContext<UserRegisteredEvent> context)
{
    var msg = context.Message;

    // Idempotency: tránh tạo trùng nếu event được deliver lại
    var existing = await _dbContext.CustomerProfiles.FindAsync(msg.UserId);
    if (existing != null) return;

    // Tạo bản sao CustomerProfile trong OrderDb (Materialized View)
    var profile = new CustomerProfile
    {
        Id           = msg.UserId,       // "aaa-111-..." (cùng ID với AuthService)
        FullName     = "nguyenvana",
        Email        = "a@gmail.com",
        LastUpdatedAt = DateTime.UtcNow
    };

    _dbContext.CustomerProfiles.Add(profile);
    await _dbContext.SaveChangesAsync(); // → lưu vào OrderDb
}
```

> **Mục đích:** Khi tạo đơn hàng, OrderService cần biết tên thật của khách để điền vào `Order.CustomerName`
> mà không cần gọi HTTP sang AuthService.

## 🛒 FLOW 2 — Đặt hàng (BankTransfer)

**Ví dụ:** Nguyễn Văn A đặt 1 Áo (200k) + 1 Quần (300k), phí ship 30k → tổng 530k, thanh toán BankTransfer.

**Trigger:** `POST /api/orders`

---

### Bước 1 — Client gọi API

```
POST http://localhost:8080/api/orders
Headers: Authorization: Bearer <jwt_token>
Body:
{
  "customerPhone": "0901234567",
  "shippingAddress": "123 Lê Lợi, Q1",
  "paymentMethod": 1,        // 1 = BankTransfer
  "shippingFee": 30000,
  "items": [
    { "productId": "prd-001", "productName": "Áo", "quantity": 1, "unitPrice": 200000 },
    { "productId": "prd-002", "productName": "Quần", "quantity": 1, "unitPrice": 300000 }
  ]
}
```

---

### Bước 2 — OrdersController tiếp nhận

**File:** `OrderService/Controllers/OrdersController.cs`

```csharp
[HttpPost]
public async Task<ActionResult<OrderResponse>> CreateOrder([FromBody] CreateOrderRequest request)
{
    Guid? customerId = null;

    // Đọc X-User-Id do Ocelot inject từ JWT (sau khi verify)
    var userIdHeader = Request.Headers["X-User-Id"].FirstOrDefault();
    if (Guid.TryParse(userIdHeader, out var fromHeader))
        customerId = fromHeader;  // "aaa-111-..."

    var result = await _orderService.CreateOrderAsync(request, customerId);
    return Ok(result);
}
```

---

### Bước 3 — OrderServiceImpl.CreateOrderAsync()

**File:** `OrderService/Services/OrderServiceImpl.cs`

```csharp
public async Task<OrderResponse> CreateOrderAsync(CreateOrderRequest request, Guid? customerId)
{
    // Lấy tên thật từ CustomerProfile (Materialized View) để không cần gọi AuthService
    var profile = await _dbContext.Set<CustomerProfile>().FindAsync(customerId.Value);
    string finalCustomerName = profile?.FullName ?? request.CustomerName;
    // → finalCustomerName = "nguyenvana"

    var newOrder = new Order
    {
        Id            = Guid.NewGuid(),
        CustomerId    = customerId,        // "aaa-111-..."
        CustomerName  = "nguyenvana",
        PaymentMethod = PaymentMethod.BankTransfer,
        ShippingFee   = 30000,
        Status        = OrderStatus.New,   // ← bắt đầu ở New
        Items = [
            new OrderItem { ProductId="prd-001", ProductName="Áo",   Quantity=1, UnitPrice=200000 },
            new OrderItem { ProductId="prd-002", ProductName="Quần", Quantity=1, UnitPrice=300000 }
        ]
    };
    // Order.SubTotal    = 200k + 300k = 500k  (NotMapped, tính từ Items)
    // Order.TotalAmount = 500k + 30k  = 530k  (NotMapped)

    await _orderRepo.AddAsync(newOrder);
    await _orderRepo.SaveChangesAsync(); // → lưu Order + Items vào OrderDb

    // Publish OrderCreatedEvent lên RabbitMQ (qua Outbox)
    await _publishEndpoint.Publish(new OrderCreatedEvent
    {
        OrderId       = newOrder.Id,
        CustomerName  = "nguyenvana",
        TotalAmount   = 530000,
        PaymentMethod = "BankTransfer",
        Items = [
            new OrderItemInfo { ProductId="prd-001", ProductName="Áo",   Quantity=1, UnitPrice=200000 },
            new OrderItemInfo { ProductId="prd-002", ProductName="Quần", Quantity=1, UnitPrice=300000 }
        ]
    });

    return MapToResponse(newOrder);
}
```

**Ví dụ:** Khách đã chuyển khoản 530k. Admin xác nhận → đơn bắt đầu đóng gói.

**File (theo guide):** `PaymentService/Services/PaymentServiceImpl.cs`

public async Task<PaymentResponse> ConfirmPaymentAsync(Guid orderId, ConfirmPaymentRequest request)
{
    var payment = await _paymentRepo.GetEntityWithSpec(new PaymentByOrderIdSpec(orderId));
    // payment.Status = Pending, payment.Amount = 530000

    payment.Status          = PaymentStatus.Succeeded;
    payment.TransactionCode = "MB202502050001"; // Admin nhập mã GD ngân hàng
    payment.PaidAt          = DateTime.UtcNow;

    _paymentRepo.Update(payment);
    await _paymentRepo.SaveChangesAsync(); // → cập nhật PaymentDb (qua Outbox)

    // Publish PaymentSucceededEvent
    await _publishEndpoint.Publish(new PaymentSucceededEvent
    {
        OrderId       = orderId,
        CustomerId    = null,      // (cần thêm vào sau)
        Amount        = 530000,
        PaymentMethod = "BankTransfer",
        TransactionId = "MB202502050001",

}

```csharp
public async Task UpdateOrderStatusIfNewAsync(Guid orderId, OrderStatus newStatus)
{
    var order = await _orderRepo.GetEntityWithSpec(new OrderWithItemsSpec(orderId));

    if (order == null || order.Status != OrderStatus.New)
        return; // Bỏ qua nếu không phải New (tránh race condition)
    // → order.Status = New → tiếp tục

    var oldStatus = order.Status;  // New
    order.Status  = OrderStatus.Processing;
    _orderRepo.Update(order);
    await _orderRepo.SaveChangesAsync(); // → OrderDb: Status = Processing

    // Publish tiếp OrderStatusChangedEvent
    await _publishEndpoint.Publish(new OrderStatusChangedEvent
    {
        OrderId       = order.Id,
        CustomerId    = order.CustomerId,    // "aaa-111-..."
        OldStatus     = "New",
        NewStatus     = "Processing",
        TotalAmount   = 530000,
        PaymentMethod = "BankTransfer",
        Items         = [ ... ]              // Áo + Quần
    });
}
```

public async Task Consume(ConsumeContext<OrderStatusChangedEvent> context)

## 🚚 FLOW 5 — Đặt hàng COD

**Ví dụ:** Khách đặt hàng COD (trả tiền mặt khi nhận). Khác với BankTransfer ở chỗ: Admin phải tự bấm Processing trước khi xác nhận thanh toán.

---

### Thứ tự đúng cho COD

```
POST /api/orders (PaymentMethod=COD)
    → Order: New, Payment: Pending

Admin đóng gói, đi giao
PUT /api/orders/{id}/status → Processing      ← ADMIN BẤM THỦ CÔNG
    → ProductService: trừ kho

Admin giao hàng, thu tiền mặt xong
PUT /api/payments/order/{id}/confirm
    → PaymentService: Payment → Succeeded
    → PaymentSucceededEvent → OrderPaymentSucceededConsumer
        → Order.Status == Processing (không phải New) → UpdateOrderStatusIfNewAsync → RETURN NGAY
        → (OrderService không làm gì thêm — đúng!)

PUT /api/orders/{id}/status → Completed       ← ADMIN BẤM THỦ CÔNG
    → CustomerService: TotalSpent += 530000
```

---

### Tại sao `UpdateOrderStatusIfNewAsync` có guard `order.Status != New`?

**File:** `OrderService/Services/OrderServiceImpl.cs`

```csharp
public async Task UpdateOrderStatusIfNewAsync(Guid orderId, OrderStatus newStatus)
{
    var order = await _orderRepo.GetEntityWithSpec(...);

    if (order == null || order.Status != OrderStatus.New)
        return; // ← COD: đơn đang Processing → return ngay, không làm gì
    //         ← BankTransfer: đơn đang New → tiếp tục đổi sang Processing
```

Guard này đảm bảo:

- **BankTransfer:** Khi PaymentSucceeded đến, đơn còn ở New → chuyển Processing ✅
- **COD:** Khi PaymentSucceeded đến, đơn đã Processing → bỏ qua ✅
- **Edge case:** Event đến trễ sau khi đơn Completed → bỏ qua, không đổi ngược ✅

| New → Processing       | PaymentSucceeded (BankTransfer) hoặc Admin | **Trừ kho**          | No-op             | —              |
| Processing → Completed | Admin                                      | No-op                | **TotalSpent +=** | —              |
| New → Cancelled        | Admin                                      | No-op (kho chưa trừ) | No-op             | —              |
| Processing → Cancelled | Admin                                      | **Hoàn kho**         | No-op             | —              |
| Completed → Cancelled  | Admin                                      | No-op (thủ công)     | **TotalSpent -=** | —              |

## 🔁 Sơ đồ tổng quan event flow

```
[AuthService]
  RegisterAsync() ──────────────────────────── UserRegisteredEvent ──▶ OrderService (tạo CustomerProfile)
                                                                    └──▶ CustomerService (tạo Customer)

[OrderService]
  CreateOrderAsync() ───────────────────────── OrderCreatedEvent ──▶ PaymentService (tạo Payment Pending)

  UpdateOrderStatusAsync() ─────────────────── OrderStatusChangedEvent ──▶ ProductService (trừ/hoàn kho)
  UpdateOrderStatusIfNewAsync()                                        └──▶ CustomerService (TotalSpent)

[PaymentService] (chưa implement)
  ConfirmPaymentAsync() ────────────────────── PaymentSucceededEvent ──▶ OrderService (New→Processing)
  MarkPaymentFailedAsync() ─────────────────── PaymentFailedEvent    ──▶ (OrderService xử lý nếu cần)
```

## ⚡ Outbox Pattern — Tại sao không mất event?

```
Không có Outbox (nguy hiểm):
  SaveChangesAsync() → ✅ lưu DB
  Publish()          → ❌ RabbitMQ down → EVENT MẤT VĨNH VIỄN

Có Outbox (an toàn):
  Publish()          → ✅ lưu vào bảng OutboxMessages (cùng DB)
  SaveChangesAsync() → ✅ lưu DB + OutboxMessages TRONG 1 TRANSACTION
  [Background Worker] → ✅ đọc OutboxMessages, gửi lên RabbitMQ (retry nếu cần)
```

Services có Outbox: **AuthService** ✅, **OrderService** ✅
Services cần Outbox (theo guide): **PaymentService** 🔴
Services không cần Outbox: **ProductService**, **CustomerService**, **ReportService** (chỉ consume)
